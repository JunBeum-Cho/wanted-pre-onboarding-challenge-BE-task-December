**(1) 동기와 비동기 프로그래밍에 대한 차이점을 설명해주세요.**

동기(Synchronous: 동시에 일어나는)

- 요청과 결과가 동시에 일어남을 의미한다.

- 요청을 하면 시간이 얼마나 걸리던 응답을 받아야 다음 요청을 처리할 수 있다.

비동기(Asynchronous: 동시에 일어나지 않는)

- 요청과 결과가 동시에 일어나지 않는다.

- 요청에 따른 응답을 받는 시간동안 다른 요청에 대한 처리도 가능하다.

동기 방식은 설계가 매우 간단하고 직관적이지만 하나의 요청을 처리할 때까지 아무것도 할 수 없다는 단점이 있고,
비동기 방식은 동기보다 복잡하지만 결과가 주어지는데 시간이 걸리더라도 다른 요청을 수행 할 수 있으니 효율적인 사용이 가능하다는 장점이 있다.

**(2) 블로킹과 논블로킹의 차이점을 설명해주세요.**

멀티 스레딩,   I/O 등에서 사용되는 개념이며, 함수의 리턴 시점과 제어권과 관련있다.

블로킹(Blocking)
- 제어권이 호출한 함수에서 호출된 함수에게 넘어가 작업이 모두 끝난 후 반환된다.
- 작업이 완료된 후 새로운 작업을 수행할 수 있다.

논블로킹(Non-Blocking)
- 제어권이 계속 호출한 함수에 있어 작업의 완료여부와 관계없이 새로운 작업을 수행할 수 있다.

**(3) 본인이 주로 사용하는 언어에서 비동기 프로그래밍을 사용하는 방법을 설명해주세요.**

python3.4에서 asyncio가 표준 라이브러리로 추가되고, python3.5부터 async, await가 문법으로 채택 되면서,
언어 자체적으로 비동기 프로그래밍이 가능하다.

기존 def 키워드 앞에 async 키워드를 붙이면 비동기 처리가 되며, 이러한 비동기 함수를 파이썬에서는 코루틴(coroutine)이라고 부르기도 한다.

```python
import asyncio

async def learn_synchronous():
    return "good!"
```

비동기 함수는 일반 함수와 같이 호출하면 coroutine 객체가 반환된다.
따라서 비동기 함수는 일반적으로 비동기 함수 내에서 await 키워드를 붙여 호출해야한다.

```python
async def main_async():
    await learn_synchronous()
```

**(4) 메세지 큐를 쓰는 이유에 대하여 2가지 예시를 서술해주세요.**

대용량 데이터를 처리하기 위한 배치 작업이나, 채팅 서비스, 비동기 데이터를 처리할 때 사용한다.

프로세스 단위로 처리하는 웹 요청이나 일반적인 프로그램을 사용하는 사용자가 많아지거나 데이터가 많아지면 요청에 대한 응답을 기다리는 수가 증가하다가 나중에는 대기 시간이 지연되어 서비스가 정상적으로 되지 못하는 상황이 오기 때문에 기존에 분산되어 있던 데이터 처리를 한곳에 집중하면서 메시지 브로커를 두어 필요한 프로그램에 작업을 분산시키는 것이 목적이다.

메시지 큐는 소비자(Consumer)가 실제로 메시지를 어느 시점에 가져가서 처리하는 지는 보장하지 않는다. 언젠가는 큐에 넣어둔 메시지가 소비되어 처리될 것이라고 믿는 것이다. 이러한 비동기적 특성 때문에 메시지 큐는 실패하면 치명적인 핵심 작업보다는 어플리케이션의 부가적인 기능에 사용하는 것이 적합하다.

예시 1) 비밀번호 찾기 이메일 발송

웹사이트의 비밀번호를 잊어 이메일을 통해 임시 비밀번호를 발급받는 상황이 있다.
이때 이메일을 큐에 넣을 수 있다. 이메일 발송 기능은 Queue의 메시지를 하나씩 소비한다.
이와 같은 방법은 메시지 큐에 들어오는 메시지가 많아지는 경우 이메일  발송 전용 인스턴스를 둠으로 확장하여 사용할 수 있다. 

예시 2) 게시글 등록 알람 기능

글을 등록하면 친구들에게 알림을 보내주는 기능이 있다.
글을 등록 하면 DB에 글을 저장하고 다시 DB에서 친구들의 정보를 가져와 알람을 보내는 일련의 과정이 필요하다.
이 과정에서오는 대기 시간을 줄이기 위해 친구들에게 알람을 보내는 기능을 Message Queue에 등록하여 처리 함으로써 단계를 줄일 수 있다.
